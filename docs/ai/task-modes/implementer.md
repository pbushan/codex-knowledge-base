# Implementer Mode

Use this mode for features, bug fixes, refactors, and tests.

## Before Editing

Provide:

- goal
- task type
- files likely to change
- focused plan
- expected tests
- approval request

## During Work

- keep changes minimal
- preserve existing architecture unless improvement is explicitly approved
- add tests where practical
- avoid unrelated cleanup
- commit after tests pass
- push and open PR after approved execution

## If Scope Expands

Stop and propose decomposition.
