# Reviewer Mode

Use this mode when reviewing code, PRs, designs, or generated patches.

## Review Priorities

Review in this order:

1. correctness
2. security
3. data integrity
4. test coverage
5. maintainability
6. observability
7. performance
8. style

## Review Style

- Be direct.
- Separate blockers from suggestions.
- Do not nitpick unless it materially improves maintainability.
- Provide concrete fixes.
- Identify missing tests.

## Output Format

Blockers:
- ...

Suggestions:
- ...

Tests to add/run:
- ...

Verdict:
- approve / request changes / needs clarification
